## 一、世界三维（锁定 · v7.4.0 · 替代原"玩家三维 + 国家九维"）

| 值 | 程序字段名 | 语义 | **初值** | 取值范围 |
|---|---|---|---|---|
| **秦之霸业 Qínzhī Bàyè** | `qin_baye` | 秦国东出 / 连横 / 单极霸权的进展 | **50** | 0–100（clamp） |
| **六国之盟 Liùguózhī Méng** | `liu_guo_meng` | 赵齐合纵 / 多极抗秦 / 联盟巩固 | **40** | 0–100（clamp） |
| **天下纷乱 Tiānxià Fēnluàn** | `tian_xia_fenluan` | 各方观望 / 局部冲突 / 失序 | **30** | 0–100（clamp） |

- **玩家三维（合纵/名望/心计）与国家九维（三国 × 国威/盟信/战心）全部移除，无任何即死**。原 §一 玩家三维表、§三 各国三维初值表、§五 6 种值死亡叙事整章删除。
- 玩家打牌、面谈立场、君主博弈、关键事件、朝议成交/威胁——**一切数值后果统一落在这三个值上**（程序入口 `State.apply_world_delta`，越界钳制到 0–100）。
- 三维互相**不完全互斥**（乱世里也可能有局部联盟）；代表"天下大势"而非"某国状态"。
- `stance_scores`（hezong/neutral/qin，MBTI 答题 + 打牌方向累计）**保留**，仅用于终局评语个性化（§2.3），**不参与结局阈值判定**。

## 二、终局判定（锁定 · v7.4.0）

### 2.1 阈值

```
第 6 回合结束 → check_ending()
  ├─ [1] 六国之盟 ≥ 70 且 秦之霸业 ≤ 60
  │      → 【合纵却秦 alliance_victory】
  ├─ [2] 秦之霸业 ≥ 70 且 六国之盟 ≤ 40
  │      → 【连横破盟 lianheng_victory】
  ├─ [3] 天下纷乱 ≥ 65（且不满足 [1]/[2]）
  │      → 【天下大乱 chaos】（v7.4.0 新增结局）
  └─ [4] 其余
         → 【纵横未决 undecided】
```

> **实施备注 2（阈值放宽）**：合纵却秦的秦之霸业阈值由 RFC-003 原案 `≤ 50` 放宽为 **`≤ 60`**——按 RFC-003-A §十 推演，即使带上调参，推合纵最优路径终局秦之霸业仍 ≈59 > 50，原阈值不可达；放宽到 60 后可达且仍需近乎全程推合纵。

### 2.2 终局名 ↔ 英文枚举（程序 / endings.json 实际值）

| 中文终局名 | 程序枚举 |
|---|---|
| 合纵却秦 | `alliance_victory` |
| 连横破盟 | `lianheng_victory` |
| 天下大乱 | `chaos`（v7.4.0 新增） |
| 纵横未决 | `undecided` |

> **注意**：RFC-003/003-A 草案中的 `hezong_success` / `lianheng_break` **未被采用**——程序与 `endings.json` 沿用 v7.3.x 既有的 `alliance_victory` / `lianheng_victory` 枚举。C18 评语库与程序枚举以本表为准（P1-9 别名统一原则继续有效）。

### 2.3 "天下大乱"结局文本 + 评语个性化（与 data/endings.json 一致）

```json
"chaos": {
    "title": "天下大乱",
    "text": "六国合纵未成，秦亦未能东出。各方观望，边境摩擦不断，百姓流离。这一年过去了，天下大势未明，乱世才刚刚开始。"
}
```

`check_ending()` 返回 `{type, detail, mbti_type}`：`detail` 由世界三维阈值决定（§2.1）；`mbti_type` 由 `judge_stance()` 取 `stance_scores` 最大者（严格大于才判派系，平局归 neutral），用于 `stance_review` 三派系 × 四结局的评语个性化。`stance_review` 三派系均已补 `chaos` 字段（hezong「你最不愿看到的结局」/ neutral「你不押注，所以没输；但你也没赢」/ qin「统一大业要等下一代」）。

### 2.4 即死结局移除

原 §五 **6 种值死亡叙事**（疑为秦间 / 合纵成·身先死 / 无声而终 / 木秀于林 / 无子可落 / 棋局反噬）随玩家三维**整章删除**；`endings.json` 的 `death` 节点已删；`check_death()` / `_check_death_and_react()` 已从代码移除。游戏中途不再有任何死亡分支，叙事张力全部由第 6 回合的四结局承载。

### 4.2 召见面谈：立场 × 君主动作结算（锁定 · 12 动作 × 3 立场 = 36 条 · 按君主分组）

面谈**无成功率、无掷骰**（v7.3.5 起沿用），不直接结算——由面谈**收场结果**决定走哪条分支：

- 玩家以三立场表态（A/B/C 预设按钮或自由输入）→ 进入多轮辩论（心证机制，详见 C13/C17）。
- **收场三分支**（心证 ≥ +2 采纳 / −1 ~ +1 自决 / ≤ −2 拒绝）：
  - **采纳** → 按下表**玩家立场**分支结算世界数值；
  - **自决** → 按下表**中立**分支结算；
  - **拒绝** → 君主坚定**全额执行原动作**（按 §4.3 君主博弈表结算）。
- **自由输入**在收场时按关键词判读立场（"降/投降"类归中立；合纵词→推合纵；连横词→推亲秦；未命中→中立）。**实施备注 4**：仲裁层对未知立场一律按"中立"分支兜底。
- 面谈完成（无论收场结果）**+1 张情报牌**（面谈摘要）。

> **实施备注 3（按君主分组）**：结算表按 秦/赵/齐 分组建键——"备战蓄力（秦）/ 备战固境（赵）"同映射 `prepare`、"骑墙观望（赵）/ 观望渔利（齐）"同映射 `observation`，语义不同，单层表会撞键。若 LLM 给出的动作不在该君主动作集内，跨君主全表兜底查同名动作。

## 五、成功率判定（锁定 · v7.4.0 · RFC-003 §5.2 方案 A）

```
行动成功率 = clamp(base_rate + 消耗情报牌数 × 5%, 5%, 95%)
掷骰 → 成功 / 失败
```

| 行动 | base_rate | 满值成功率（5 张情报牌） |
|---|---|---|
| 游说 | 45% | 70% |
| 传信 | 45% | 70% |
| 许诺 | 50% | 75% |
| 离间 | 40% | 65% |
| 刺探 | 55% | 80% |
| 召见面谈 | — | 无成功率（无掷骰） |

- **属性加成移除**：原"名望 × 0.3"等玩家三维加成随玩家三维删除；`cards.json` 的 `scale_attr` / `scale_coef` 置空弃用。base_rate 微调补偿（游说 40→45 / 离间 35→40 / 刺探 50→55 / 许诺 45→50 / 传信 40→45）。
- **情报牌加成（v7.3.3 沿用）**：每张 +5%，任意数量，受 clamp(5%, 95%) 封顶；消耗后移出情报牌区。**面谈无成功率，不适用情报牌加成**。
