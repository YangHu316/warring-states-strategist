### 1.2 召见面谈（**v7.3.7 重设计 · RFC-002 · P0-2/3/4 修复**）

面谈**不是二值裁决**——无成功率、无掷骰、无采纳/拒绝、**不影响玩家三维**（无 1.5 倍、无名望 −5）。但玩家的表态会**影响国家三维（全局数据）**。

**核心设计原则**（**v7.3.7 重大变更**）：**3 立场必须有差异化结果**。原 v7.3.5 机制下"推合纵 = 推亲秦 = 背书执行原 proposed_action"是核心逻辑悖论——若君主 proposed_action 是"离间赵齐"，玩家推合纵却执行离间，与立场语义冲突。

#### 1.2.1 流程（v7.3.7）

- 君主 Agent 简述与他国的博弈，**提出一个具体行动意图**（= `proposed_action`），以一句问句征询玩家。
- 玩家输入简短 prompt 表达**推合纵 / 中立 / 推亲秦**（与打牌时一致的三立场）。
- 程序读 `stance_aware_actions`（见 C12）：根据 `player_stance` 选取分支：
  - **`if_hezong`**（玩家推合纵背书）→ 执行"合纵侧"结果
  - **`if_qin`**（玩家推亲秦背书）→ 执行"亲秦侧"结果
  - **`if_neutral`**（玩家中立）→ 君主自决（原 proposed_action 效果）
- LLM 输出 schema：`{opening, proposed_action, player_stance, response, resolved}`（详见 11b §四）。

#### 1.2.2 `stance_aware_actions` 设计（C12 困局的新字段）

每个困局 JSON 增加 `stance_aware_actions` 三分支：

```json
{
  "stance_aware_actions": {
    "if_hezong": {
      "action_name": "劝阻 + 反制",
      "effect_summary": "原动作被反制 + 合纵侧 +",
      "national_delta": "..." // 程序按 C01 §3.2 实际计算
    },
    "if_qin": {
      "action_name": "协助执行",
      "effect_summary": "原动作被强化",
      "national_delta": "..."
    },
    "if_neutral": {
      "action_name": "君主按原 proposed_action 自决",
      "effect_summary": "原 proposed_action 效果",
      "national_delta": "..."
    }
  }
}
```

#### 1.2.3 推合纵 / 推亲秦 效果差异原则（v7.3.7）

| 维度 | 推合纵背书 | 推亲秦背书 | 中立/君主自决 |
|---|---|---|---|
| **原 proposed_action** | 反制 / 削弱 | 强化 | 原样执行 |
| **合纵相关国家** | 赵/齐盟信 +3~+5（合纵巩固）| 赵/齐盟信 −3~−8（合纵削弱）| 君主原意图（按 C01 §3.2） |
| **秦相关** | 秦国威 / 战心 涨幅降低或不变 | 秦国威 / 战心 涨幅强化 | 君主原意图 |
| **玩家三维** | 不变 | 不变 | 不变 |
| **君主记忆** | `player_leans_hezong` | `player_leans_qin` | `player_neutral` |

#### 1.2.4 示例（v7.3.7）

**D-Q1 · 试探合纵虚实**（原 proposed_action = 遣使离间）：

| 玩家立场 | 效果 | 国家三维 |
|---|---|---|
| 推合纵 | 玩家劝阻离间 → 离间被反制 | 赵齐盟信 +3（合纵巩固） |
| 推亲秦 | 玩家主动帮秦离间 | 赵齐盟信 −10（离间强化） |
| 中立 | 君主自决（按原 proposed_action）| 赵齐盟信 −8（按 §3.2） |

> **重要**：v7.3.7 修订后，**3 立场有完全不同的国家三维结果**——这是修复 P0-2/3/4 的核心。

### 1.3 面谈超时降级（锁定 · v7.3.5）

面谈 LLM 超时 8s → 关键词匹配判读玩家立场：
- 检测"合""纵""盟""抗秦""联六国"等 → **推合纵** → 执行 proposed_action
- 检测"横""连横""亲秦"等 → **推亲秦** → 执行 proposed_action
- 都不命中 → **中立** → 君主自决（默认执行）

> - 终局 A 阈值：`赵盟信 ≥ 55 / 齐盟信 ≥ 55 / 秦国威 ≤ 75` → `赵盟信 ≥ 45 / 齐盟信 ≥ 35 / 秦国威 ≤ 80`

## 四、二值判定机制（锁定）

```
行动成功率 = clamp(基础成功率 + 对应数值 × 加成系数 + 消耗情报牌数 × 5%, 5%, 95%)
掷骰 → 成功 / 失败
```

| 行动 | 基础成功率 | 加成来源 | 满值成功率（未计情报牌） |
|---|---|---|---|
| 游说 | 40% | 名望 × 0.3 | 70% |
| 离间 | 35% | 心计 × 0.3 | 65% |
| 刺探 | 50% | 心计 × 0.25 | 75% |
| 许诺 | 45% | 名望 × 0.3 | 75% |
| 传信 | 40% | 合纵 × 0.3 | 70% |

> **召见面谈无成功率**（v7.3.5）：不掷骰、不判成败、不影响玩家三维，改为玩家以**三立场**（推合纵/中立/推亲秦）表态——推合纵/推亲秦=背书则影响**国家三维**，中立=君主自决（详见 §1.2）。

> **情报牌加成（v7.3.3）**：打主动行动牌前可消耗任意数量情报牌，**每张 +5% 成功率**，无硬上限，受 `clamp(5%, 95%)` 封顶。消耗后情报牌移出情报牌区（不可复用）。详见 [C02 §五](C02-卡牌效果与方向选择-锁定表.md)。**面谈无成功率，不适用情报牌加成**。
