> **编号**：RFC-003
> **状态**：✅ 已实施（v7.4.0 · 2026-07-07 · 一次性切换，无兼容期；实施细节与偏差见 RFC-003-A 实施备注）
> **提交**：2026-07-03 策划端
> **影响**：跨 10+ 文件架构重设计
> **前置**：C01-数值系统-锁定表.md（v7.3.7 D1 锁定版）
> **优先级**：P0（设计层冲突，影响结局判定正确性）

## 一、动机与问题诊断

### 1.1 现存设计冲突

当前游戏并行维护两套数值系统，存在结构性冲突：

**系统 A：玩家三维**（`player_attrs = {hezong, mingwang, xinji}`）
- 来源：玩家打牌 / 面谈结算
- 用途：`check_death()` 即死结局（任一值 0 或 100）+ 卡牌成功率加成（`scale_attr`）
- 共 3 维

**系统 B：国家三维**（`country_attrs = {qin/zhao/qi} × {guowei, mengxin, zhanxin}`）
- 来源：君主 Agent 行动 + 玩家面谈 stance 修饰
- 用途：`check_ending()` 结局判定（合纵胜/连横胜/未决）+ LLM ctx
- 共 9 维

**系统 C：stance_scores**（`{hezong, neutral, qin}`）
- 来源：玩家打牌方向累计 + MBTI 答题
- 用途：`judge_stance()` 玩家派系标签（仅结局文本展示）

### 1.2 已暴露的具体冲突

**冲突案例**（玩家反馈）：

> "我在劝说齐国保持中立后，齐王采纳了我的建议，齐国和秦国的盟信都提升了，然而齐国的盟信数值在结局的考量里是合纵的一环。也就是说，我虽然劝说了中立，但是却推动了合纵的局势。"

**根因分析**：

1. **盟信不区分方向**：`country_attrs.qi.mengxin` 既包含"齐对赵的盟信"（合纵方向），也包含"齐对秦的盟信"（连横方向）。结局判定 `qi.mengxin >= 55` 一律视为合纵胜条件，与玩家意图脱节。
2. **中立立场仍走原动作**：`settle_proposed_action_with_stance(stance="中立")` 直接调 `settle_proposed_action`（原动作），原动作可能是 `seek_alliance`（求盟）→ 仍加双方 mengxin → 推高合纵胜条件。
3. **结局只看数值不看玩家立场**：`check_ending()` 完全忽略 `stance_scores`，玩家即使 8 次选"中立"，结局仍由 `country_attrs` 单方面决定。

### 1.3 为什么修补无效

历次修订（RFC-001 / RFC-002）都在现有架构上做阈值调整：

- RFC-001：放宽终局 A 阈值（55/55/75 → 45/35/80）
- RFC-002：秦国威漂移 +5→+3、stance_aware_actions 重设计

**根本问题未解决**：盟信的方向性语义缺失 + 玩家立场与数值脱钩。任何阈值微调都无法消除"玩家意图 ↔ 数值变化 ↔ 结局判定"三者之间的语义错位。

## 二、设计目标

1. **消除方向性歧义**：盟信等数值应明确表达"对谁"或"哪一阵营"
2. **玩家立场 ↔ 结局一致**：玩家选"中立"→ 数值变化不应推动合纵/连横任一方
3. **简化数值维度**：9 维国家三维 + 3 维玩家三维 → 3 维世界数值，降低玩家认知负担 + LLM ctx 复杂度
4. **取消即死结局**：玩家三维与即死结局移除（你已确认）
5. **保留派系标签**：MBTI stance_scores 保留，仅用于结局评语个性化

---

## 三、新设计：三项世界数值

### 3.1 三维定义

| 值 | 语义 | 初值 | 取值范围 |
|---|---|---|---|
| **秦之霸业 Qínzhī Bàyè** | 秦国东出 / 连横 / 单极霸权的进展 | 50 | 0–100 |
| **六国之盟 Liùguózhī Méng** | 赵齐合纵 / 多极抗秦 / 联盟巩固 | 40 | 0–100 |
| **天下纷乱 Tiānxià Fēnluàn** | 各方观望 / 局部冲突 / 失序 | 30 | 0–100 |

**设计原则**：
- 三维互相**不完全互斥**（乱世里也可能有局部联盟，但合纵未成气候）
- 三维代表"天下大势"，而非"某国状态" —— 玩家不再需要分别记忆 9 个国家数值
- 玩家行动 / 君主 Agent 行动都直接改变这三项世界数值

### 3.2 结局判定（替代 §3.1 旧阈值）

```
第 6 回合结束 → check_ending()
  ├─ [1] 六国之盟 ≥ 70 且 秦之霸业 ≤ 50
  │      → 【合纵却秦 hezong_success】
  ├─ [2] 秦之霸业 ≥ 70 且 六国之盟 ≤ 40
  │      → 【连横破盟 lianheng_break】
  ├─ [3] 天下纷乱 ≥ 65（且不满足 [1]/[2]）
  │      → 【天下大乱 chaos】（新增结局）
  └─ [4] 其余
         → 【纵横未决 undecided】
```

**与 stance_scores 的综合**：
- 结局判定（合纵胜/连横胜/乱世/未决）由世界三维主导
- `judge_stance()` 返回的玩家派系（hezong/neutral/qin）用于结局评语个性化（已存在的 `stance_review` 字典）
- 两者**不冲突** —— 玩家可以是"合纵派"但合纵未成（undecided），结局评语会体现这种"理想未竟"的张力

### 3.3 漂移参数（替代旧 §二）

每 2 回合（回合 2/4 开始时，6 不漂移）：

| 漂移项 | 量 | 语义 |
|---|---|---|
| 秦之霸业自然增长 | +4 | 秦崛起脊骨（原秦国威 +3 + 赵齐国威 +2 的合并） |
| 六国之盟自然侵蚀 | -3 | 联盟的自然衰退（原赵齐盟信 -2 的合并） |
| 天下纷乱自然增长 | +2 | 局势持续失序 |

## 四、动作 → 世界数值映射

### 4.1 卡牌行动（替代 cards.json 旧 on_success 字段）

| 卡牌 | 方向 | 旧影响（玩家三维） | 新影响（世界数值） |
|---|---|---|---|
| 游说 | 推合纵 | hezong +4 | **六国之盟 +5, 秦之霸业 -2** |
| 游说 | 推亲秦 | hezong -4 | **秦之霸业 +5, 六国之盟 -3** |
| 游说 | 中立 | mingwang +5, xinji +3 | **天下纷乱 -3**（调停缓和局势） |
| 传信 | 利好合纵 | hezong +3 | **六国之盟 +3** |
| 传信 | 利好连横 | hezong -3 | **秦之霸业 +3** |
| 传信 | 中立 | xinji +5 | **天下纷乱 -2** |
| 许诺 | 援助 | xinji +6, mingwang +3 | **六国之盟 +2, 天下纷乱 -1** |
| 许诺 | 结盟 | hezong +2, mingwang +5 | **六国之盟 +4** |
| 许诺 | 中立 | xinji +5, mingwang +2 | **天下纷乱 -2** |
| 离间 | (无方向) | hezong -3（高盟信时） | **六国之盟 -5, 天下纷乱 +3** |
| 刺探 | (无方向) | xinji +3 | **天下纷乱 +1**（探听增加紧张） |
| 召见面谈 | - | 不影响玩家三维 | 见 §4.2 |

**失败通用**：`天下纷乱 +2`（行动失败增加失序感，替代原"名望 -3"）

### 4.2 召见面谈 stance_aware_actions（替代 arbiter.gd 的 settle_proposed_action_with_stance）

每个君主 proposed_action 按玩家 stance 三分支结算世界数值：

**示例：秦·遣使离间（proposed_action = alienate）**

| 玩家 stance | 效果 | 世界数值变化 |
|---|---|---|
| 推合纵 | 玩家劝阻 + 反制离间 | **六国之盟 +3, 天下纷乱 -2** |
| 推亲秦 | 玩家主动帮秦离间 | **六国之盟 -8, 秦之霸业 +3** |
| 中立 | 君主自决（执行原离间） | **六国之盟 -5, 天下纷乱 +2** |

**示例：赵·求盟联齐（proposed_action = seek_alliance）**

| 玩家 stance | 效果 | 世界数值变化 |
|---|---|---|
| 推合纵 | 玩家背书 + 强化求盟 | **六国之盟 +6, 秦之霸业 -2** |
| 推亲秦 | 玩家劝赵放弃求盟 | **秦之霸业 +3, 六国之盟 -3** |
| 中立 | 君主自决 | **六国之盟 +3**（半幅） |

**示例：齐·观望渔利（proposed_action = observation）**

| 玩家 stance | 效果 | 世界数值变化 |
|---|---|---|
| 推合纵 | 劝齐从观望转向合纵 | **六国之盟 +4, 天下纷乱 -2** |
| 推亲秦 | 齐坐实观望渔利 | **秦之霸业 +2, 天下纷乱 +2** |
| 中立 | 君主自决 | **天下纷乱 +2**（观望加剧失序） |

**关键改进**：
- "中立"不再是"原动作原样执行"，而是有明确的中立效果（通常降低纷乱或半幅执行）
- 玩家选"中立" → 不会推动合纵也不会推动连横，符合"中立"语义

### 5.2 卡牌成功率新机制

原 `rate = base_rate + player_attrs[scale_attr] × scale_coef + intel_bonus`

**新方案 A（推荐）**：固定基础成功率 + 情报牌加成
```
rate = clamp(base_rate + intel_bonus × 5, 5, 95)
```
- 移除 player_attrs 依赖
- 卡牌成功率完全由基础值 + 情报牌决定
- `base_rate` 微调（游说 40→45，离间 35→40，刺探 50→55，许诺 45→50，传信 40→45）补偿失去 player_attrs 加成

**新方案 B（备选）**：用世界数值替代 player_attrs 加成
```
rate = clamp(base_rate + world_attr_value × scale_coef + intel_bonus × 5, 5, 95)
```
- 游说 / 许诺 → scale on `六国之盟`（玩家越推合纵，越擅长此牌）
- 传信 → scale on `秦之霸业`（连横方传信越有效）
- 离间 / 刺探 → scale on `天下纷乱`（越乱越容易刺探）
- 优点：玩家三维的概念以"世界数值"形式延续
- 缺点：世界数值本身由玩家行为推动，存在循环依赖（推合纵 → 六国之盟上升 → 游说成功率上升 → 更容易推合纵）

## 七、迁移策略

### 7.1 兼容期（建议）

- `state.gd` 保留 `country_attrs` 字段作为 deprecated 别名，内部映射到 `world_attrs` 的派生计算（如 `country_attrs.qin.guowei = world_attrs.qin_baye`）
- `apply_country_delta` 改为转发到 `apply_world_delta`
- 兼容期 1 个版本（v7.3.11），v7.3.12 彻底删除

**优点**：程序成员可分批迁移调用点，降低一次性改坏风险
**缺点**：兼容期内存两套数据，需注意一致性

### 7.2 一次性切换

直接删除旧字段，所有调用点一次性改完。

**优点**：代码干净
**缺点**：跨 10+ 文件一次性改，bug 风险高，需要充分测试

### 7.3 推荐

**推荐方案 7.1（兼容期）**，但若程序成员评审认为改动可控，可走 7.2。

## 八、风险与开放问题

### 8.1 已知风险

1. **LLM prompt 调整**：monarch_ai.gd / dialogue.gd / agent_manager.gd 的 LLM ctx 都含 country_attrs，改为 world_attrs 后 prompt 语义变化，可能需要重新调试 LLM 输出质量
2. **平衡性重做**：所有卡牌 / 动作 delta 重新设计后，6 回合可达性需重新推演（原 C01 §3.1 推演作废）
3. **存档兼容**：若有存档功能（当前无），旧存档的 country_attrs 无法直接迁移到 world_attrs

### 8.2 待评审决策

1. **卡牌成功率机制**：§5.2 方案 A（固定基础值）还是方案 B（用世界数值加成）？
2. **chaos 结局的具体阈值**：§3.2 暂定 `天下纷乱 ≥ 65`，是否合理？需要 D3 mock 验证
3. **国家状态显示**：UI 上是否还显示每国状态（召见/决策已定）？还是只显示世界三维？
4. **君主 Agent 决策**：`_score_actions` 原依赖 `qin.guowei >= 75` 等阈值，改为 world_attrs 后阈值需重新标定
5. **briefing 文本格式**：召见前情摘要原显示"秦：国威70 盟信30 战心60"，改为"秦之霸业50 / 六国之盟40 / 天下纷乱30"是否过于抽象？

### 8.3 不在本次 RFC 范围

- LLM prompt 的具体措辞调整（由程序成员在代码迁移时同步调）
- D3 mock 50 局平衡校准（C24 任务）
