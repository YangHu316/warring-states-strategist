## 一、动机与问题诊断

v7.4.x 实测暴露三个体验问题：

1. **AI 群体智能体现不强**：君主博弈的对象是三根抽象数值条，动作没有实体后果，"秦离间赵齐"和"齐观望渔利"在玩家眼里只是数字微调。
2. **玩家缺乏反馈，漫无目的**：赶路只为触发面谈，打牌对世界的影响不可见也不可感，地图是静态背景。
3. **玩家没有压力**：不限时、无死亡、无行动成本，任何回合都可以什么都不做。

**根因**：当前世界没有"实体"，只有进度条。君主争的是数字，玩家撬的也是数字。

**本 RFC 的答案**：给世界装上实体（城池、兵力、财富），让一切博弈围绕实体展开（交易、盟约、战争），让实体变化全部可见（板块格子条、军队棋子、战报），并给玩家装上生存约束（名望命条 + 行动点）。

**RFC-003 的教训必须继承**：删玩家三维的真正原因是"三套数值语义打架、变动不可感知"，而不是"玩家数值不好"。v8 的数值分层原则：**每个数值只回答一个问题，且只有一个事实来源**。

### 1.4 验收场景（r2 起为最高设计约束）

本版的第一验收对象不是"一局玩到底"，而是**演示前十分钟**：观看者要在 3 分钟内看到 AI 出兵、地图上的军队在动、玩家一句话让军队停下或加速；10 分钟内看到一场战争的完整因果（出兵 → 斡旋 → 交战或收兵 → 城池易手 → 列国连锁反应）。一切设计决策以"这件事在前十分钟里看不看得见"为优先级排序依据。完整流程脚本见 §十四 Phase A 验收。

## 三、国力层

### 3.1 定义与初值

| 国 | 城池 | 兵力 | 财富 | 人设注解 |
|---|---|---|---|---|
| 秦 | **25** | **80** | **70** | 强兵多城，东出之志 |
| 赵 | **18** | **60** | **50** | 武而不富，首当其冲 |
| 齐 | **20** | **45** | **85** | 富而不武，待价而沽 |

- 城池：0–40，整数，全局共 63 座（守恒，只在国家间转移）。
- 兵力：0–100。财富：0–100。
- State 字段：`national: {qin: {cities, troops, wealth}, ...}`，信号 `national_changed(country)`。

### 3.2 回合再生（每回合开始结算）

| 项 | 公式 | 秦 | 赵 | 齐 |
|---|---|---|---|---|
| 兵力回复 | `+floor(城池/5)` | +5 | +3 | +4 |
| 财富增长 | `+floor(城池/4)` | +6 | +4 | +5 |

> 战争损耗需 3–5 回合恢复 → 君主不能无限开战，外交仍是主旋律。

### 3.3 资源交易（替代抽象 delta 的核心机制）

一切"许诺/利诱/成交"落地为**结构化交易**：

```
交易 = { from, to, give: {cities: n, wealth: n}, ask: 条款文本, due_round, status }
status: pending（已允诺）→ kept（已兑现）/ broken（违诺，见 §七）
```

- 朝议"应允"成约 → 生成交易（due = 当前回合 +1）。
- 到期在回合结算时自动执行：城池/财富转移 + 地图格子翻色。
- 例：「秦许齐三城换中立」→ 下回合结算时秦 25→22、齐 20→23，秦板块右侧 3 格翻为齐色。

> 盟约分两档：**通好**（朝议成交的一般约定）/ **军事同盟**（seek_alliance 达成，触发参战义务与联军反攻资格）。

## 四、战争系统（三点位行军制：军队的位置就是压力条）

### 4.1 行军流程

每对国家的隐形路径上设 **3 个点位**，军队棋子逐点推进，位置即战争进度、即倒计时、即谈判筹码：

```
宣战（军费 −20 财，战报横幅「秦王下令伐赵！」）
   ▼
【点位① 刚出兵】  棋子出现在攻方都城外
   │  战报：「秦军拔营东进」；被攻方【自动召见玩家】
   ▼  （一个节拍后自动推进）
【点位② 行进半途】 棋子至两国之间
   │  战报：「秦军过半程，邯郸告急！」；守方再次告急召见
   ▼
【点位③ 兵临城下】 棋子抵守方城下
   │  战报：「兵临城下——最后一搏！」；最后通牒窗口（一个节拍）
   ▼
交战结算（§4.2）→ 割城（格子翻色）+ 损兵 + 名望结算 + 入承诺账本
```

**推进节拍**：自由行动阶段内每 **40 秒**自动前进一个点位；回合结算时无论如何强制前进一格（防止玩家挂机拖延——**时间不等人**）。

**点位可点击**：点击棋子/点位弹军情卡（谁的军队、兵力几何、下一次推进倒计时、盟军参战状态）——刺探牌可揭示其真实兵力。

### 4.1.1 玩家操控行军（核心交互：你的话推得动军队）

面谈**军队所属君主**，心证三档收束直接映射为行军指令：

| 玩家意图 | 面谈结果 | 行军效果 |
|---|---|---|
| 劝收兵（推合纵/劝和） | 采纳 | **收兵**：棋子沿路径回撤消失，战争取消（军费不退）；攻方 2 回合内不得再对同一目标宣战 |
| 劝缓兵 | 自决（半信半疑） | **减缓**：跳过下一个节拍（点位停留时间翻倍） |
| 怂恿疾进（推亲秦/怂恿） | 采纳 | **推进**：立即前进一个点位 |
| 劝阻失败 | 拒绝 | 军队照常推进，且君主记下你的阻挠 |

面谈**守方君主**与第三国的间接操控：

| 途径 | 效果 |
|---|---|
| 劝守方割地求和 | 生成议和交易（价码见 §4.1.2）→ 攻方收兵 |
| 促成第三国军事同盟/参战 | 守方战力 +盟军×0.5 → 攻方 Agent 立即重评估：胜算不足则**自动收兵**（棋子回撤，全场可见的"合纵却敌"时刻） |
| 传信·假情报（"前方有伏兵"） | 攻方**减缓**一个节拍；被识破则加速一格 |
| 离间守方的盟约 | 守方失去盟军 → 攻方重评估：可能由缓转急 |

### 4.1.2 议和价码随点位上涨（拖延的代价可见）

守方向攻方求和的价码，由军队当前位置决定：

| 军队位置 | 议和价码 |
|---|---|
| 点位① 刚出兵 | 财 15，或玩家以名望担保的口头承诺 |
| 点位② 行进半途 | 城 1 |
| 点位③ 兵临城下 | 城 2 + 财 10 |

> 这张表是"压力迫使决断"的核心：早介入便宜，晚介入割肉，不介入挨打。价码在守方召见的困局台词中由君主明说。

### 4.2 交战结算表

```
攻方战力 = 攻方兵力 + 盟军兵力×0.5 + rand(−10, +10)
守方战力 = 守方兵力×1.2 + 盟军兵力×0.5 + rand(−10, +10)
差值 = 攻 − 守
```

| 差值 | 结果 | 割城 | 攻方损兵 | 守方损兵 |
|---|---|---|---|---|
| ≥ +30 | 攻方大胜 | 攻方夺 3 城 | −10 | −25 |
| +10 ~ +29 | 攻方小胜 | 攻方夺 2 城 | −15 | −20 |
| −9 ~ +9 | 僵持 | 无 | −15 | −15 |
| −29 ~ −10 | 攻方败退 | 无 | −25 | −10 |
| ≤ −30 | 攻方溃败 | **守方反夺 1 城** | −30 | −8 |

**推演锚点**：开局秦(80)单独攻赵(60×1.2=72)期望差 +8 → 僵持边缘。秦必须先备战或拆掉赵齐同盟才能碾压；赵齐军事同盟后守方 72+22.5=94.5 → 秦必败。**合纵真的能却秦，离间真的能亡赵**——群体智能从此有实体后果。

### 4.3 限制（防失控）

- 全局同时最多 **1 场**战争；每国每 **2 回合**至多发起 1 次。
- 收兵后同一攻方对同一目标 2 回合内不得再宣战。
- 回合结束时若军队仍在途中，战争跨回合延续（点位保留），回合结算强制推进一格。
- 城池 ≤ 8 的国家不可被继续宣战（保留残局，不做灭国——灭国留待后续版本）。
- **坐视也是选择**：玩家全程不介入，交战照常结算——不作为的后果全程可见。

## 五、玩家生存层

### 5.1 名望（单值命条 · r3 替代三国信望方案）

> r3 修订理由：三条信望需要分别结算，认知与结算负担都高。改为单值后，**玩家只盯一根命条**；"君主各自怎么看你"不再是玩家数值，而是 Agent 的记忆（心证初值修正 + 承诺账本恩怨，见 §9.3）——数值归玩家，感情归君主。

| 值 | 语义 | 初值 | 归零 | 爆表 |
|---|---|---|---|---|
| **名望 Míngwàng** | 你在列国间的声名与分量 | 50 | 「无声而终」：名望扫地，无人再纳你之言 | 「名动天下者死」：一言兴邦覆国，三王共忌 |

- 取值 0–100，**归零或爆表 → 即死**（叙事见附录 A）。
- 名望是双刃剑：它是武器（接入卡牌成功率，§十——名望高游说易），也是催命符（越会说话，离爆表越近）。

### 5.2 名望变动表（大开大合；**阳谋扬名、阴谋不显、败露重挫**）

| 来源 | 变动 |
|---|---|
| 面谈收束·采纳 | **+8**（一言兴邦，列国传颂） |
| 面谈收束·自决 | +2 |
| 面谈收束·拒绝 | **−6**（碰壁的说客） |
| 辩论中失言（shift = −2） | 当场 −2 |
| 你化解一场战争（劝退/促和/搬救兵成功） | **+15**（弭兵之名天下知） |
| 你怂恿的战争打赢 | **+10**（连横/合纵之功） |
| 你怂恿的战争打输 | **−12**（祸国之名） |
| 游说成功 / 许诺兑现 | +4 / **+8** |
| 离间成功 / 刺探成功 | **±0**（阴谋之功，不显于名） |
| 离间被识破 / 刺探被抓 | **−8 / −6**（阴谋败露，声名狼藉；该君主衔恨入账本） |
| 违诺（你担保的交易崩了） | **−15**（失信之名） |
| 怠慢召见（未响应至回合结束） | **−5**（傲慢之名；该君主衔恨入账本） |
| 立场抉择（每回合开场事件） | ±4（见 §5.5） |

> 不对称设计：光明正大的成功涨名望；阴谋成功不涨（隐秘之功无人知晓），其收益走别处（情报、拆盟、战局），败露则重挫——**走阳道攒命，走阴道换利**，两条玩法路线由此分化。

### 5.3 边缘加速与自救台阶（伴君如伴虎的手感）

- **边缘加速**：名望 ≥ 80 → 每回合自动 +3（列国争相传颂，把你推向功高震主）；≤ 20 → 每回合 −3（墙倒众人推）。中段（21–79）不漂移。
- **自救台阶**（防止即死挫败感，各触发一次）：
  - 首次跌至 ≤ 15：触发「毛遂自荐」事件——本回合内任一次面谈达成"采纳" → 额外 +10；
  - 首次升至 ≥ 85：触发「功高」事件——可花 1 行动点「自污」（称病、辞赏、散财，主动 −15）。

### 5.4 行动点（AP）

- 每回合 **4 点**：赶路 1 / 打牌 1 / 面谈（含响应召见）1 / 挑战已定决策 1。
- 问卷抉择、读密信、查看情报牌不耗点。弃牌不耗点。
- 点数用尽仍可观察博弈与推进回合；不可存点。
- 顶栏显示 `行动 ◆◆◆◇`。
- 设计意图：三国若同时召见，你**必然怠慢至少一国**（响应一次 = 赶路1+面谈1 = 2 点）——取舍即压力。

### 5.5 立场问卷 → 立场抉择

保留每回合 1 题、6 题题库、三选一、立场累计判定（结局评语用），但每题改为**有代价的公开事件**：

> 例 Q1「秦使当廷问你：合纵连横，先生持何论？」
> A 直言合纵 → 名望 +4，秦王衔恨（入账本）｜ B 顾左右而言他 → 名望 −2 ｜ C 附和连横 → 名望 +4，赵王衔恨（入账本）

- 30 秒倒计时保留，超时默认 B（名望 −2——**沉默也有代价**）。
- 公开表态的政治代价走承诺账本（君主恩怨 → 影响后续心证初值），不占玩家数值。
- C06 题库需按此格式重写（策划文档任务）。

## 九、Agent 决策改造

### 9.1 mock 决策（_score_actions 输入升级）

决策输入从大势三维改为：**兵力比 / 城池比 / 财富 / 盟约状态 / 失信标记 / 战争状态**。示例权重：

- 秦·宣战：`我兵 ≥ 目标兵×1.3 且 目标无军事同盟` → 权重 1.8；目标有同盟 → 0.2
- 秦·离间：`赵齐有军事同盟或 pending 盟约谈判` → 1.6（先拆盟再打）
- 赵·求盟：`秦兵/我兵 ≥ 1.2` → 1.5；`齐有失信标记` → ×0.5
- 齐·待价：`存在 pending 出价` → 1.8，出价按 `城×8 + 财×0.5` 折算比价
- 齐·趁火打劫：`某国兵 < 40 且我兵 ≥ 50` → 1.4

### 9.2 LLM prompt 升级

- 「天下大势」段之后新增「列国国力」表（城/兵/财 + 盟约 + 战争状态 + 失信标记）。
- 决策输出扩展：`action_type` 枚举加入 `declare_war / call_ally / accept_deal / reject_deal`；交易类动作要求结构化输出 `deal: {give: {cities, wealth}, ask: "条款"}`（仲裁器校验范围：城 ≤ 3，财 ≤ 25）。
- 决策规则追加：「宣战前先算兵力比与盟约——打不赢的仗不打，能用三城买来的中立不用刀兵」。

### 9.3 心证初值挂接国力与恩怨（r3：承接"君主信任"的语义）

面谈初始心证在现有盟约修正上追加两类修正：
- **国力修正**：劝弱国抗强国（兵力比 < 0.7）→ 推合纵初值再 −1（"先生要孤以卵击石？"）；目标国刚战败 → 任何立场初值 +1（病急乱投医，更听得进话）。
- **恩怨记忆**（读承诺账本）：该君主上次纳你之谏 → +1；上次拒你之谏、被你阻挠战事、或有"衔恨"记录（阴谋败露/怠慢/公开与其作对）→ −1。**君主对你的信任是 Agent 的记忆，不占玩家数值位**——这是 r3 单值化后三国态度差异的实现方式。

## 十、卡牌重定义

成功率公式：`rate = base + 情报牌×5 + (名望 − 50)×0.3`，clamp 5–95。（名望高游说易——但名望越高离爆表越近，**武器即催命符**）

| 牌 | v8 语义 | 成功效果 | 失败 |
|---|---|---|---|
| 游说（选方向） | 影响君主决策倾向 | 名望 +4；目标下一轮对应动作权重 ×1.5（推合纵→求盟/求援；推亲秦→接受秦交易） | 名望 −3 |
| 传信（选方向） | **情报武器**（战争介入核心） | 向目标注入兵力情报：真（校正认知）/ 假（目标对某国兵力 ±15 认知偏差，一回合）；入账本 | 被识破：名望 −5，该君主衔恨入账 |
| 许诺 | **以你的名望担保交易** | 促成一笔 pending 交易立即成约（担保人：你）；兑现 → 名望 +8；违诺 → 名望 −15 | 名望 −3 |
| 离间（时机窗保留） | 拆交易/拆盟约 | 目标国的 pending 交易/盟约谈判作废（阴谋不显名，名望 ±0）；对有失信记录的两国 +15 成功率 | 被识破：名望 −8，该君主衔恨入账 |
| 刺探 | 窥意图 + **真实兵力**（对抗假情报） | 精确情报牌：目标意图 + 城/兵/财真值（名望 ±0） | 细作被抓：名望 −6，该君主衔恨入账 |

- 牌堆构成与抽牌节奏不变（回合初补 2 张，上限 5）。
- 打牌前弹窗预览保留，新增名望变动预览与战争时机提示。

| MBTI/立场累计与 stance_review | 保留；问卷升级为抉择（§5.5） |

## 十二、6 回合推演（设计推算）

**亲秦线**（玩家全程助秦）：R1 离间赵齐盟谈成功 → R2 帮秦利诱齐（割2城换中立，秦25→23）→ R3 秦备战两轮（兵80→92）宣战赵 → R4 交战：92 vs 72 → 大胜夺3城（秦26）→ R5 再宣战 → R6 结算小胜夺2城（秦28）+ 齐中立到期归还2城质押（28+2=30）……**约 30–33 城，连横破盟可达但需全程投入**（阈值 32 允许一次失手，D 期推演校准）。

**合纵线**：R2 玩家促成赵齐军事同盟（面谈+许诺担保）→ R3 秦宣战赵、玩家不拆盟 → R4 交战 80 vs 94.5 → 秦溃败（−30 兵，赵反夺1城，秦24）→ R5 赵 call_ally 联军反攻：105×?… 联军攻方 60+22.5=82.5 vs 秦 50×1.2=60 → 大胜夺3城（秦21）→ R6 若再胜一场 → 秦 ≤ 20 且同盟存续 → **合纵却秦，需两场胜仗，紧但可达**。

**大乱线**：玩家假情报怂恿三场战争 + 担保交易两次违诺 → 纷乱 = 3×15 + 2×8 = 61 + 毁约 → ≥ 65 → **天下大乱**。

> 三线均可达，数值以 D 期 mock 批量推演（≥30 局）为准校调，本表为首版锚点。

## 十四、分期实施与验收（r2 重排：演示优先，每期结束游戏可玩）

### Phase A · 战争可视化闭环（demo 核心，约 3 天）
范围：城池/兵力最小集（财富先用常量占位）+ 城池格子条 + 宣战→三点位行军→交战→割城 + 面谈操控行军（收兵/减缓/推进）+ 议和价码 + 军队棋子与点位 UI + 战报横幅 + 自动召见 + **首局第一场战争脚本化**（R1 关键事件固定触发「秦军拔营伐赵」）。

**验收 = 黄金十分钟脚本**（演示流程，一步都不能断）：

```
0:00 开局 → 立场抉择 → 关键事件横幅「秦军拔营伐赵！」（首局必发）
0:30 秦军棋子出现在点位①，战报横幅；赵国红叹号自动召见
2:00 玩家赶赴邯郸面谈赵王——价码当面摆开：搬齐国救兵 / 割 1 城求和 / 另想办法
4:00 路线甲（合纵）：转赴临淄促成赵齐军事同盟 → 秦军 Agent 重评估胜算不足
     → 棋子当众回撤（"合纵却敌"时刻）
     路线乙（连横）：面谈秦王怂恿疾进 → 棋子跳进一格 → 兵临城下 → 交战
     → 赵割 2 城，格子逐格翻色 + 战报
6:00 朝议连锁反应（三国 Agent 引述此战互相放话/成交）+ 恩怨入承诺账本
8:00 第二回合开始：君主带着这一仗的记忆决策——观看者已看懂全部循环
```

### Phase B · 国力完善（约 1.5 天）
财富与回合再生 + 交易系统完整化（利诱/待价/违诺判定）+ agent mock 权重全面接国力 + 大势派生公式替换 world_attrs 直接加减。
**验收**：一笔「三城换中立」从朝议成交到格子翻色全链路走通；违诺公示并影响该国后续决策。

### Phase C · 生存层（约 1 天，r3 单值化后工作量下降）
名望单值 + 即死/爆表 + 边缘加速 + 自救台阶 + 行动点 + 怠慢结算 + 问卷抉择化 + 心证恩怨修正（§9.3）。
**验收**：AP 不足以响应全部召见，取舍真实发生；名望命条三档警示可感；至少体验到一次自救台阶。

### Phase D · 收束（P2 · 老板验收后再排期）
终局实体锚点（§6.2）+ endings 死亡叙事 + mock 批量推演 ≥30 局 + 调参 + C 系列文档同步。
**在此之前暂沿用 v7.4 大势终局判定**——派生三维照常喂给现有 check_ending，零改动可工作。

## 十五、风险与开放问题

**风险**
1. LLM 交易结构化输出的校验成本（对策：仲裁器硬校验 + 超范围回退 mock 报价表）。
2. 战争与面谈的时序竞态（集结期玩家正在面谈时交战结算——对策：交战结算延迟至玩家离开面谈场景）。
3. 数值再平衡工作量大（对策：D 期批量 mock 推演驱动，锚点先行）。

**待评审决策**
1. **行军节拍 40 秒**是否合适？（30s 更紧张 / 60s 更从容；演示场合建议偏快）
2. **首局脚本化战争**固定「秦伐赵」，还是三选一随机（秦伐赵/秦伐齐/齐赵摩擦）？固定版演示更稳，建议固定。
3. AP = 4 是否合适？（3 更紧张 / 5 更宽松）
4. 名望爆表即死的"自救台阶"是否仅一次？（两次更宽松，但会稀释伴君如伴虎）
5. 玩家要不要有独立"资财"（打牌花钱）？本版暂不加，AP 已承担行动约束。
6. （P2）连横破盟阈值 32 城 vs 30 城；灭国是否开放——随 Phase D 一并评审。
