## 二、地图系统

### 2.1 地图元素

| 元素 | 描述 |
|---|---|
| **国家节点** | 秦（左）/ 赵（中）/ 齐（右），显示名称 + 三维数值条 |
| **玩家位置** | 图标在节点间移动，有路径线和动画 |
| **使节动画** | Agent 博弈时，小图标沿路径在两国间移动 |
| **国家状态标签** | `谈判中·第1轮` / `谈判中·第2轮` / **`召见`** / `决策已定` / `已处理` |
| **路径** | 秦↔赵、赵↔齐（秦↔齐不直连） |

### 2.2 国家状态机

```
    自由行动阶段开始
          │
          ▼
     ┌───────────┐
     │ 谈判中     │  第 1 轮：Agent 评估局势，选择初始动作
     │ ·第1轮     │
     └─────┬─────┘
           │ 第 1 轮完成
           ▼
     ┌───────────┐
     │ 谈判中     │  第 2 轮：基于第1轮结果+玩家干预，做出最终决策
     │ ·第2轮     │
     └─────┬─────┘
           │ 第 2 轮完成
           ▼
     ┌───────────┐     ┌───────────┐
     │   召见     │ 或  │ 决策已定   │
     │ (等玩家)   │     │ (已自决)   │
     └─────┬─────┘     └─────┬─────┘
           │ 玩家面谈完成    │ 玩家打牌挑战
           │                │  → 成功 → 召见
           │                │  → 失败 → 名望 −3
           ▼                ▼
     ┌───────────┐
     │  已处理    │
     └───────────┘
```

- "召见"标签有**视觉高亮**（脉冲光晕），提示玩家可前往打字。
- "决策已定"标签为灰色——君主已有定见，但玩家仍可访问（打牌挑战）。

### 2.3 地图交互

- **点击国家**：不在该国 → 开始赶路。已在该国且状态为"召见" → 进入面谈。已在该国且状态为"决策已定" → 弹出决策摘要（不打牌不弹面谈）。
- **hover 国家**：显示三维数值 + 当前博弈轮次 + Agent 最近动作摘要。
- **hover 使节**：显示"秦使赴赵：张仪携离间之议"等说明。

## 三、赶路系统

### 3.1 移动规则

- 点击目标国家 → 玩家图标沿路径移动（动画约 3–5 秒，不受 Agent 博弈状态影响）。
- 移动中可打牌。
- 抵达时若该国为"召见" → 自动弹出面谈界面。
- 抵达时若该国为"决策已定" → 弹出决策摘要 + 玩家可选择"打牌挑战"或"离开"。
- 可在移动中途改变目的地。

### 3.2 移动策略维度

| 策略 | 说明 |
|---|---|
| **等 vs 走** | 看到赵即将召见但齐已召见 → 先去齐还是等赵？ |
| **先打牌再出发** | 出发前先刺探目标国，了解其意图再决定是否动身 |
| **赶往谈判中的国家** | 在第 1 轮和第 2 轮之间赶到，打出卡牌干预 Agent 的第 2 轮决策 |
| **折返** | 已在去赵路上，事件流显示"秦正向齐发起军事施压" → 折返救局 |

## 四、召见系统（打字面谈 · 不限时 · 立场表达 · 无成功率）

### 4.1 进入条件

玩家抵达处于"召见"状态的国家 → 自动触发面谈界面。

### 4.2 面谈界面

```
┌──────────────────────────────────────────┐
│  召见 · 赵王                               │
│                                          │
│  [赵王立绘/头像]                           │
│                                          │
│  ┌─ 召见前情 ─────────────────────┐     │
│  │ · 秦第1轮：向齐许诺割三城       │     │
│  │ · 赵第1轮：求盟齐王             │     │
│  │ · 齐第2轮：待价而沽（已决）     │     │
│  │ · 平原君力主联齐，廉颇主备战    │     │
│  └────────────────────────────────┘     │
│                                          │
│  "孤正与齐王商议合纵之事，齐王遣使来说，  │
│   若赵不先表态，他不敢动。               │
│   足下以为孤该当如何？"                  │
│                                          │
│  ┌──────────────────────────────────┐   │
│  │ 简短说说你的看法…                  │   │
│  │                                    │   │
│  └──────────────────────────────────┘   │
│                                          │
│  [发送]                                  │
│  你已持有情报牌 2 张                      │
└──────────────────────────────────────────┘
```

> **无"预计成功率"**（v7.3.3）：面谈不是判定行动，界面不显示成功率，也无情报牌加成入口。

### 4.3 面谈流程（不限时 · 无成功率）

1. **君主开场**：Agent 基于该国博弈结果，简述"正在与谁博弈、遇到什么僵局"（≤ 80 字），**提出一个行动意图**（= 其某个博弈动作），最后一句问句征询玩家意见。
2. **玩家表态**：输入一段**简短 prompt**（推荐 ≤ 80 字）表达赞同/反对/中立。**不限时**。
3. **君主回应 + 结算**：Agent 判读玩家态度 →
   - **赞同** → 提议动作对应的**国家三维**变动生效（见 [03 §2.2](03-数值系统.md)）。
   - **反对** → 搁置不生效，记入君主记忆。
   - **中立** → 君主自决（默认执行）。
4. **面谈结束**：该国状态 → 已处理 + 玩家获得 1 张情报牌（面谈摘要）。

### 4.4 打字设计约束

- 玩家可输入中英文混合，不做语义格式校验。
- **面谈无成功率、无掷骰、无采纳/拒绝、不影响玩家三维**（无 1.5 倍、无名望 −5）；玩家态度改为影响**国家三维**（见 §4.5 结算）。
- LLM 超时 8 秒 → 回退关键词匹配判读玩家态度（赞同/反对/中立，默认中立）。
- **面谈不限时**：无倒计时，无催促。

### 4.5 LLM 输出 schema（v7.3.5 · 立场采集 + 国家三维结算）

```
LLM 输出（schema 约束）：
{
  "opening": "≤ 80 字开场：简述与他国博弈 + 提出行动意图 + 一句问句",
  "proposed_action": "君主提议的博弈动作（见其可用动作枚举）",
  "player_stance": "赞同" | "反对" | "中立",  // Agent 对玩家态度的判读
  "response": "≤ 60 字君主听取后的回话",
  "resolved": true | false                      // 是否执行 proposed_action
}
```

结算（仲裁器）：
- `player_stance = 赞同` → `resolved = true` → proposed_action 对应的**国家三维**变动生效（见 [03 §2.2](03-数值系统.md)）。
- `player_stance = 反对` → `resolved = false` → 搁置，记入君主记忆。
- `player_stance = 中立` → 君主自决（默认 `resolved = true`）。

> **关键设计**：面谈不判成败、不改玩家三维；玩家对君主提议的态度直接左右**国家三维（全局数据）**走向——赞同即为君主的行动"背书"。

### 4.6 关键词匹配（mock 兜底）

超时 8s 后回退到 mock，判读玩家对君主提议的态度：
- 检测"可""好""善""准""依""行"等 → `player_stance = 赞同` → 执行 proposed_action
- 检测"不""否""缓""慎""莫""且""待"等 → `player_stance = 反对` → 搁置
- 都不命中 → `player_stance = 中立` → 君主自决

## 五、回合推进机制

### 5.1 [进入下回合] 按钮

位于状态栏右侧。三种状态：

| 按钮状态 | 条件 | 含义 |
|---|---|---|
| **灰色 · 不可点击** | 仍有 Agent 博弈未完成（某对仍在第 1/2 轮） | "还有国家在谈判中" |
| **亮起 · 可点击** | 所有三对博弈已完成（全部进入"召见"或"决策已定"） | 可以推进到下月 |
| **亮起 + 有未处理召见提示** | 有召见未被玩家处理 | 提示"还有召见未处理，确定进入下回合？"→ 玩家可选"再等等"或"确定进入" |

### 5.2 回合结束

玩家点击"进入下回合" → 数值结算 → 即死检查（玩家三维 0/100 → 死亡）→ 未死则进入下一大回合。

## 六、博弈可视化三层

### 第一层：地图实时视觉

| 可见元素 | 刷新时机 | 体现的 AI 博弈 |
|---|---|---|
| 国家三维数值条 | 每轮 Agent 博弈结算后 | 各国实力/信用/战意实时变化 |
| 使节移动动画 | Agent 选择"遣使/施压/求盟"时 | 直观看到谁在接触谁 |
| 国家状态标签 | 博弈轮次切换 + 完成后 | "谈判中·第N轮" → "召见" / "决策已定" |

### 第二层：事件流滚动

每条事件 = Agent 博弈动作摘要：

```
[秦→齐] 第1轮：秦王遣张仪入齐，以割三城为条件换取齐王观望…
[赵→齐] 第1轮：赵王遣平原君向齐求盟——"赵齐唇齿相依"…
[齐] 第1轮：齐王在秦赵之间权衡——孟尝君力劝谨慎…
[秦→赵] 第2轮：秦王向赵施压——若不中立则率先伐赵…
[赵] 第2轮：赵王决策中——平原君坚持联齐，廉颇主张备战…
[赵] 赵王决定求盟齐王——[召见]
[齐] 齐王第2轮完成——待价而沽，两头不得罪——[决策已定]
```

### 第三层：召见前情摘要 + 情报牌

- 面谈时展示该国本轮博弈的完整脉络。
- 情报牌补充已获取的详细情报。

## 七、决策已定国家的玩家交互（v7.3 新增）

### 7.1 抵达"决策已定"国家

```
┌──────────────────────────────────────┐
│  抵达 · 齐国 [决策已定]                 │
│                                      │
│  「齐王决定待价而沽，闭门观望——        │
│    看来需要打破他的算盘」              │
│                                      │
│  [打牌挑战]   [离开]                  │
└──────────────────────────────────────┘
```

- **打牌挑战**：选择手牌 + 方向 → 视为对决策的挑战。
  - 成功 → 推翻决策（该国状态 → 召见）→ 玩家可前往面谈深谈。
  - 失败 → 名望 −3 + 该国状态保持"决策已定"。
- **离开**：直接返回地图。
- **不允许打字面谈**（无困局可解）。

### 7.2 决策已定国家是否值得玩家前往

| 玩家立场 | 是否值得 |
|---|---|
| 偏合纵 | 值得——可挑战齐王的"待价而沽" |
| 偏连横 | 值得——可挑战赵王的"求盟齐" |
| 中立 | 看情况——可打探信息 |

## 八、MVP 简化范围

- 地图动画：使节移动用点+虚线，不做复杂 2D。
- 事件流：纯文本滚动。
- 打字输入：Godot 原生 TextEdit。
- 国家数值条：横向色块+数字。
- 情报牌：纯文本卡，排列在侧栏。**打牌确认框内提供"消耗情报牌加成"多选入口**（v7.3.3）——勾选 N 张 → 预计成功率实时 +N×5%（受 95% 封顶），确认打出后被勾选的情报牌移出情报牌区。
- 决策摘要：弹出 modal，文字 + 选项按钮。
