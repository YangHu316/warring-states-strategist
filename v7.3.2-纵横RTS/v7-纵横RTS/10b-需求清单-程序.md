## D3（全天 8h · 核心：LLM 接入 + 三立场计分 + 联调）

### 上午（4h）

- [ ] **ENG-10（0–1.5h）** LLM 接入层
  - **做什么**：
    - `LLMClient.gd`：HTTPRequest 调用 LLM API。
    - `AgentContextBuilder.gd` 加载 [11b §一](11b-角色设计稿-程序.md) 的 **8 模块 prompt 结构**（角色定位 / 性格画像 / 核心利益 / 可用动作 / 近臣倾向 / 记忆滚动摘要 / 决策规则 / 输出 schema），按君主组装 system prompt。
    - 关键事件 + 关键事件 implication 注入（在"6. 记忆"之前）。
    - JSON schema 约束 + 解析 + 校验。
    - 超时 5s → 降级 mock。
  - **验收物**：单 Agent LLM 调用跑通 + 3 君主 8 模块 prompt 解析日志。
  - **依赖**：C16（3 君主 LLM prompt）+ 11b 文档。

- [ ] **ENG-11（1.5–3h）** 三君主同步博弈调度（≤ 2 轮/对）
  - **做什么**：
    - 自由行动阶段开始 → 三君主第 1 轮 LLM 并行请求。
    - 第 1 轮全部完成 → 自动进入第 2 轮（+1s）。
    - 玩家打牌 → 第 2 轮立即触发（+0.5s）。
    - 第 2 轮完成 → "召见"或"决策已定"。
    - 超时 5s → 该 Agent 回退 mock。
  - **验收物**：3 Agent 同步博弈跑通。
  - **依赖**：ENG-10。

- [ ] **ENG-12（3–4h）** 召见打字 LLM 管线（不限时 · 立场结算国家三维 · 无成功率 · v7.3.5）
  - **做什么**：
    - 玩家打字（不限时，简短）→ 发送 → LLM 请求。
    - 接收 `{opening, proposed_action, player_stance, response, resolved}`（v7.3.5 · 无二值裁决）。
    - **无成功率判定**：`player_stance=赞同`（resolved=true）→ 按 [03 §2.2] 动作影响表结算 `proposed_action` 对应的**国家三维**变动；`反对`→ 搁置；`中立`→ 君主自决。
    - 反馈：君主 `response` 回话 + 国家三维变化 + 1 张情报牌（面谈摘要）。**不改玩家三维**。
    - 超时 8s → 降级 mock：按关键词判读赞同/反对/中立（默认中立）。
  - **验收物**：召见 LLM 真实交互跑通。
  - **依赖**：C17（召见 prompt）。

- [ ] **ENG-22（7.5–8h · D3 下午最后 0.5h）** Agent 性格调优循环（**核心补漏**）
  - **为什么必须有**：11b / 04 文档定义了三君主的"雄猜/犹疑/渔利"性格 + 5 近臣权重 + 200 字记忆滚动。但**仅靠 ENG-10 的"注入 prompt"不等于 agent 真按性格决策**——LLM 可能忽略性格画像、按通用策略回答。本任务就是**让性格真的体现**。
  - **做什么（4 步循环，跑 2 轮）**：
    1. **基线收集**：用 ENG-10/11/12 当前 prompt 各跑 3 局（每局 6 回合），记录**每轮 3 君主的 `{action, reason, confidence}`**，导出为 `tune_baseline.json`。
    2. **性格一致性检查表**（每君主 5 项 × 3 君主 = 15 项）：
       - 秦王嬴稷（雄猜）：① 合纵迹象出现时**必须**先破（遣使离间） ② 多疑时倾向军事施压 ③ 决策 confidence 多 ≥ 7 ④ 优先选较弱/犹豫方施压 ⑤ 行动频率高
       - 赵王赵何（犹疑）：① 默认动作偏 `求盟联齐` 或 `骑墙观望` ② 听到合纵合理建议时动心（confidence 偏高） ③ 第 2 轮常进入"召见" ④ confidence 多 ≤ 6 ⑤ 行动频率中
       - 齐王田地（渔利）：① 默认 `观望渔利` ② 收到连横利诱才 `待价而沽` ③ 看到某方衰弱才 `趁火打劫` ④ 常出"决策已定" ⑤ 行动频率低
    3. **逐项打勾 / 不达标**：打勾的保留 prompt；不达标的分析原因（性格画像字数不足？决策规则优先级不够清晰？近臣低语被忽略？记忆影响太弱？），**修改 [11b](11b-角色设计稿-程序.md) 文档对应模块的措辞**。
    4. **重跑验证**：修改后再跑 3 局，对比基线 → 至少 12/15 项达标（80%）→ 通过；否则再迭代一轮。
  - **调优维度**（按不达标原因选择性调整）：
    - **性格画像**：把"雄猜"等抽象词**翻译为具体行为约束**（如"雄猜"→ "当六国有合纵迹象时，你必须先破"），放在 prompt 第 2 模块头部。
    - **决策规则**：把 11b §2.1–2.3 的"决策规则"模块从 6 行扩到 10–15 行，**显式给优先级**（如"动作优先级：离间 > 利诱 > 军事 > 蓄力"）。
    - **近臣低语**：从"低声说"改为**注入到决策规则尾部**（"在选 `连横利诱` 前，张仪在你耳边说：'齐王贪利，许以三城可定'——这使你更倾向选它"），权重描述更显眼。
    - **记忆滚动**：把"200 字滚动摘要"中的**关键决策结果高亮**（如"**上一轮你用离间失败了——赵齐盟信反而上升**"），让 LLM 看到。
    - **schema 收紧**：在 `expected_effect` 字段强制要求 LLM 写"基于我 [性格] 的判断……"作为理由，强迫它引用性格。
  - **降级兜底**（调优超过 2 轮仍不达标时）：
    - 方案 A：把 prompt 进一步**结构化为分步选择题**（"步骤 1：是否察觉合纵迹象？Y/N → 步骤 2：选动作……"），减少 LLM 自由发挥。
    - 方案 B：**增加 mock 兜底频率**——LLM 性格偏离时，仲裁器用 mock 决策表结果覆盖。
    - 方案 C：若 D3 实在来不及，**只调优秦王**（最容易识别"雄猜"），赵/齐 暂以 mock 为主 → D4 补调。
  - **验收物**：
    - `tune_baseline.json`（3 局决策日志）
    - `tune_checklist.md`（15 项打勾记录，标红不达标项）
    - `tune_iter1.json`（调优后 3 局决策日志，对比基线）
    - **调优后的 11b 文档 patch**（标黄改动的措辞）
    - 12/15 项达标签字
  - **依赖**：ENG-10/11/12 + 11b 文档 + C16 prompt 终稿。
  - **执行人**：程序 + 策划（C16 prompt 作者）**联调**，程序负责 prompt 注入和日志，策划负责判断决策是否"像秦王/赵王/齐王"。
  - **风险**：见 [11b §十一](11b-角色设计稿-程序.md) ——"LLM 决策不基于性格"的对策是本任务的核心。

**D3 验收线**：
- LLM 跑完一局；同步博弈可视化；召见打字真实交互；三立场评语匹配正确。
- 9 条评语匹配。
- 断网无缝回退。
- **3 君主 12/15 项性格一致性格检查达标**（ENG-22）。

## 砍线触发

| 触发 | 砍什么 | 保住什么 |
|---|---|---|
| D2 晚地图+回合推进没跑通 | 自由行动阶段退化为顺序三段，砍同步博弈和事件流 | 抽牌+三维即死+mock 召见+方向选择 |
| D3 上午 LLM 接入受阻 | 砍到每对仅 1 轮；事件流退化为回合末总结 | 召见+数值即死+三立场 |
| 打字面谈 LLM 不稳/太重 | 召见退化为预制选项（赞同/反对/中立） | 卡牌+三维+三立场 |
| 三立场计分来不及 | 砍到 4 回合 4 题 | 必交 4 题 |
| 9 评语来不及 | 砍到 3 立场 × 1 终局 = 3 条（仅合纵却秦） | 必交 3 条 |
| 全面落后 | 手牌 4 张上限；砍传信牌；终局 2 种；君主 2 个 | 核心闭环 |
| ENG-22 调优第二轮仍不达标 | 砍到只调秦王 1 个，赵/齐 回退 mock；prompt 改为分步结构化 | 秦王 5/5 项达标 + 核心闭环 |

## 边界处理 checklist（D4 必查）
