### 2.2 P0-2/3/4: 面谈机制核心逻辑重设计（v7.3.7 stance_aware_actions）

**问题**：
- v7.3.5 旧：推合纵 = 推亲秦 = 背书执行 proposed_action
- 若君主 proposed_action = 离间赵齐，玩家推合纵却执行离间 → **核心逻辑悖论**
- 推合纵 vs 推亲秦 效果几乎无差异

**修订方案**：

每个 C12 困局 JSON 增加 `stance_aware_actions` 三分支：

```json
{
  "stance_aware_actions": {
    "if_hezong": { "national_delta": "赵齐盟信 +3" },
    "if_qin": { "national_delta": "赵齐盟信 −10" },
    "if_neutral": { "national_delta": "赵齐盟信 −8" }
  }
}
```

**设计原则**：
| 维度 | 推合纵背书 | 推亲秦背书 | 中立/自决 |
|---|---|---|---|
| 原 proposed_action | **反制** / 削弱 | **强化** | 原样执行 |
| 合纵相关国家 | 盟信 +3~+5 | 盟信 −3~−8 | 君主原意图 |
| 秦相关 | 涨幅降低或不变 | 涨幅强化 | 君主原意图 |
| 玩家三维 | 不变 | 不变 | 不变 |

**影响文件**：
- `策划交付物/D1-策划交付物-20260630/C01-数值系统-锁定表.md`（§1.2 重写）
- `策划交付物/D2-策划交付物-20260702/C12-召见困局prompt模板.md`（**14 困局全部增加 stance_aware_actions**）
- `策划交付物/D2-策划交付物-20260702/C13-mock召见立场判读规则.md`（结算流程 + JSON v7.3.7）
- `v7-纵横RTS/03-数值系统.md`（§1.1/§3.1 重写）
- `策划交付物/AI_INTEGRATION_GUIDE.md`（§1.2/§7.5/§7.6 同步）

### 2.6 衍生修订：秦国威漂移 +5 → +3

**问题**：
- v7.3.6 推演发现秦国威过高（88 > 80），终局 A 几乎不可达
- 玩家在 6 轮内无论怎么推合纵，秦国威都因 drift +5×2 + 备战 +6 = 16 增长至 86

**修订方案 A（推荐）**：秦国威漂移 +5 → +3
- 6 轮 drift 总增长 +6（2 次触发）
- 秦王 2 次备战蓄力 +6
- 总计 70+6+6 = 82（玩家推合纵 6 次面谈 if_hezong 反制 2 → 80 终值）

**影响文件**：
- `策划交付物/D1-策划交付物-20260630/C01-数值系统-锁定表.md`（§二漂移参数）
- `v7-纵横RTS/03-数值系统.md`（§1.2 漂移参数）
