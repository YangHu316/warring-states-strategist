## 一、设计原则

1. **每条 ≤ 30 字**（含标点）—— 注入到 LLM prompt 第 5 模块"近臣倾向"前
2. **5 个近臣 × 5 条台词 = 25 条**
3. **每近臣的 5 条覆盖不同场景**：
   - 1 条开场（他登场时说什么）
   - 2 条劝告（在君主犹豫时说什么）
   - 1 条威胁（局势紧张时说什么）
   - 1 条收场（事成或事败时说什么）
4. **台词必须符合近臣性格**（张仪狡黠 / 魏冉粗犷 / 平原君儒雅 / 廉颇刚直 / 孟尝君老谋深算）

## 八、注入逻辑

| 触发 | 注入方式 |
|---|---|
| 君主 LLM 调用前 | 抽 1 条匹配当前 scene 的近臣台词 → 注入到 prompt 第 5 模块"近臣倾向"前 |
| mock 决策表分支匹配 | 抽 1 条匹配当前 scene 的近臣台词 → 注入到 `minister_weights.low_voice` 字段 |
| 召见 Agent | 抽 1 条 matching 君主 + 近臣 → 注入到 prompt 第 5 模块 |

> **示例**（秦王 LLM 调用 · 当前 scene = "劝告（连横）"）：
> ```
> # 5. 近臣倾向
> 「齐王反复无主，许以重利则齐必动。」——张仪在你耳边低语。
> - 张仪：连横利诱 confidence +2
> - 魏冉：军事施压 confidence +1
> ```
