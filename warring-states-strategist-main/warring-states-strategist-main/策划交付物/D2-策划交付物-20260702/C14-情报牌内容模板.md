## 一、设计原则

### 1.1 情报牌双重身份（v7.3.3 升级）

| 用途 | 字段 | 说明 |
|---|---|---|
| **信息认知** | `text`（情报内容 ≤ 60 字） | 玩家随时回顾 |
| **成功率加成** | `can_consume_for_bonus` / `bonus_value` | 打牌前可消耗，每张 +5% |

## 十、字段名一致性（与 C05 契约对齐）

| 字段 | 来源 | 类型 |
|---|---|---|
| `card_id` | C05 §2.1 IntelCard | String |
| `source_action` | C05 §2.1 | String ∈ `{PERSUADE, ALIENATE, SPY, PROMISE, DISPATCH, AUDIENCE}` |
| `source_direction` | C05 §2.1 | String ∈ `{PUSH_HEZONG, PUSH_QIN, NEUTRAL_ADVICE, null}` |
| `source_target` | C05 §2.1 | String ∈ `{qin, zhao, qi, null}` |
| `text` | C05 §2.1 | String ≤ 60 字 |
| `round_acquired_range` | C05 §2.1 | String ∈ `{any, "1-3", "4-6"}` |
| `can_consume_for_bonus` | v7.3.3 新增 | bool |
| `bonus_value` | v7.3.3 新增 | int（默认 5） |
| `display_priority` | v7.3.5 新增 | int（1=最先显示） |

## 十二、待确认项 ⚠️

1. **51 条是否过多**——情报牌库越丰富，玩家读取时越有"信息通胀"感。D2/D3 试玩验证。
2. **召见情报牌 9 条是否需要**——v7.3.5 新增，因召见成为稳定 +1 情报牌来源。**保留。**
3. **`source_target = null`** 用于跨君主/无目标情报（如 INTEL_DISPATCH_NEUTRAL_*）——程序需支持 null 字段。
4. **情报牌 spawn_rules 触发逻辑**——目前是"成功 → 抽 1 张"，是否改为"成功 → 抽 1 张 + 1 张 follow-up"？**D3 校准**
