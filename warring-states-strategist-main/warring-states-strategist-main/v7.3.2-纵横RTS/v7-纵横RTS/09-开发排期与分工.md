# 09 · 开发排期与分工（v7.3.1 · 3.5 天）

> 框架：**3.5 天**（D1 全天 + D2 全天 + D3 全天 + D4 半天）· 3 人（策划 + 程序 + 美术）· 单局 6 回合。
> ⚠️ v7.3.1 关键变更：三立场问卷（每回合 1 题），评语从 48 条减为 9 条，题库从 12 题减为 6 题。
> 工时总览：3.5 × 8h × 3 人 ≈ **84 人时**。必须严格 mock 先行 / 最小闭环先行 / 不重排。

## 一、完成定义（DoD）

1 个 Godot 包：6 回合制跑通；每回合开始三立场问卷 1 题 + 关键事件生成 + 抽行动牌；自由行动阶段（不限时，手动推进）；Agent 博弈 ≤ 2 轮/对，围绕关键事件展开；二值判定打牌 + 方向选择；召见打字面谈（不限时）+ LLM 立场结算（无成功率 · 影响国家三维）；情报牌后置获取；3 立场 × 3 终局 = 9 条评语结局；三维即死 + 态势终局；降级链完整；可现场演示 + 录屏。

## 二、范围围栏（明确不做）

- ❌ 打字输入建议/自动补全（用原生输入框）。
- ❌ 复杂 2D 地图动画（使节移动用虚线+点）。
- ❌ 第四国/更多 Agent。
- ❌ 语音输入 / BGM / SFX / 复盘回放。
- ❌ 角色立绘的复杂多表情（**仅 1 张主立绘 + 1 张召见背景**）。
- ❌ MBTI 16 型复杂判别（**改用 3 立场简单判定**）。
- ✅ 保留：卡牌策划（含方向选择）、三维即死、主线脊骨、mock 离线可用、三立场问卷、博弈可视化。

## 七、MVP vs Stretch

- **MVP（必交）**：6 回合制 + 三立场问卷 + 关键事件 + 自由行动 + 地图赶路 + 召见打字 + 卡牌方向选择 + 三君主同步博弈（LLM+mock）+ 7 牌堆 + 6 题题库 + 9 评语 + 6 种值死亡 + 3 种终局 + 降级链 + 录屏。
- **核心目标**：玩家能感知"AI 在博弈" + "我被系统识别" + "再来一局换策略"。
- **Stretch**：传信自由输入；使节 2D 动画；地图缩放；BGM。

## 九、风险

1. **3.5 天工期极紧**——D2 必须 mock 跑通最小闭环。D1 不排重活，只锁设计+打底。
2. **三立场问卷每回合弹出**——新增 UI 组件（modal 弹窗）+ 计分逻辑 + 立场匹配 → 程序 D2 必须完成，策划 D2 必须出 6 题题库。**9 评语工作量大幅缩减**，D2 上午可完成。
3. **9 评语 + 6 题 + 12 月剧本**——策划 D2 工作量大幅缩减（48→9 评语；12→6 题），D2 上午出题库+评语初稿，下午出总体剧本。
4. **3 君主 + 5 近臣 = 8 个角色立绘**——美术 D2 必须出 8 张立绘草图，D3 出终版。**D1 风格打样 + 草图必须当日完成**。
5. **卡牌方向选择 UI**——D2 必出，D3 接 LLM 时验证。
6. **LLM 同步博弈对延迟敏感**——对策：三君主每轮并行发送 LLM 请求；超时 5s 回退 mock。
7. **玩家打字质量不可控**——对策：召见时 Agent prompt 中明确"若无法理解玩家意图，请礼貌追问"；立场判读（推合纵/中立/推亲秦）有关键词兜底。
8. **不限时可能导致回合过长**——对策：Agent 博弈 2 轮完成后自然收敛，召见数量有限；D4 试玩观察平均单回合耗时。
9. **LLM agent 性格表现不稳定**（**v7.3.1 新增风险**）——11b 文档定义的三君主"雄猜/犹疑/渔利"性格，可能因 LLM 通用策略干扰而无法体现。对策：**ENG-22 性格调优循环**（D3 下午 7.5–8h 基线 3 局 + D4 上午 0.5–1.5h 调优），15 项一致性格检查至少 12/15 达标；不达标时回退到结构化 prompt（分步选择题）或增加 mock 兜底频率。详见 [10b · ENG-22](10b-需求清单-程序.md)。
