# 04 · Agent 架构（v7.3 · 自由行动同步博弈）

## 一、核心变更：从"响应式"到"同步式"

v6 中 Agent 仅在玩家打牌时被动反应。v7 中 Agent 在自由行动阶段的**整段时间都在主动博弈**——与玩家行动**完全并行**。

```
┌─ 自由行动阶段开始 ─────────────────────────────┐
│                                                 │
│  ┌─ 秦 Agent ──┐ ┌─ 赵 Agent ──┐ ┌─ 齐 Agent ──┐│
│  │ 持续运行      │ │ 持续运行     │ │ 持续运行     ││
│  │ 评估局势      │ │ 评估局势     │ │ 评估局势     ││
│  │ 选择对手      │ │ 选择对手     │ │ 选择对手     ││
│  │ 执行博弈动作  │ │ 执行博弈动作 │ │ 执行博弈动作 ││
│  │ 输出结果      │ │ 输出结果     │ │ 输出结果     ││
│  └──────────────┘ └──────────────┘ └──────────────┘│
│           ↓               ↓               ↓       │
│  ┌─ 代码仲裁器 ──────────────────────────────────┐│
│  │ 校验 Agent 输出 · 结算国家数值 · 更新地图状态  ││
│  │ 写入事件流 · 触发召见标签 · 生成后续事件        ││
│  └──────────────────────────────────────────────┘│
│                                                 │
│  ┌─ 玩家（并行）────────────────────────────────┐│
│  │ 赶路 · 打牌 · 面谈 · 观察博弈动态              ││
│  └──────────────────────────────────────────────┘│
└─────────────────────────────────────────────────┘
```

**铁律**：Agent 不直接改数值。Agent 输出 {目标, 动作, 意图} → 代码仲裁器查表结算数值 → 更新国家三维 + 地图状态 + 事件流。

## 二、同步博弈的运行方式

自由行动阶段不限时。每位君主 Agent 在阶段开始后即进行**最多 2 轮**自主决策，耗时因 LLM 延迟而异。2 轮后自动停止。

### 2.1 轮次上限机制（防永久博弈）

```
每回合，每对国家（秦↔赵、秦↔齐、赵↔齐）最多 2 轮博弈。

第 1 轮（自动启动，阶段开始 +1s）：
  各 Agent 接收当前世界状态 → 评估局势 → 选择初始博弈动作
  → 仲裁器结算 → 写入事件流 → 国家状态 = "谈判中·第1轮"

第 2 轮（自动启动，第 1 轮全部完成后 +1s，或玩家打牌后 +0.5s）：
  Agent 基于第 1 轮结果 + 任何玩家干预（打牌）→ 做出最终决策
  → 仲裁器结算 → 写入事件流
  → 国家状态 = "召见"（有待玩家解决的困局） 或 "决策已定"（君主已有定见）

2 轮后该国博弈锁定——不再有新动作。
```

- 每对国家独立计数。秦可能比赵齐先完成 2 轮。
- 若玩家在 Agent 博弈中途打出卡牌 → Agent 在下一轮裁决中**必须考虑**该干预（玩家打牌触发第 2 轮提前启动）。
- 当所有三对博弈均完成 → "[进入下回合]"按钮亮起。

### 2.2 轮次触发时机（v7.3 明确）

| 触发 | 时机 |
|---|---|
| **第 1 轮** | 自由行动阶段开始 +1 秒（让地图先渲染一帧） |
| **第 2 轮** | 三对第 1 轮全部完成后 +1 秒；或任意玩家打牌 +0.5 秒（打断重评估） |
| **轮次锁定** | 每对第 2 轮完成后即停 |

> **解决 v7.2 模糊点**：第 2 轮不是"等玩家打牌"——是"自动启动但可被玩家打牌打断重评估"。

### 2.3 每位君主每轮的决策循环

```
1. 接收"当前世界状态包"：
   - 自己国家的三维数值
   - 其他两国的三维数值
   - 上一轮博弈的结果（如果有）
   - 近臣的当前倾向权重
   - 本回合关键事件
   - 玩家本轮打牌的影响（若有——例如被打离间牌、被传信）

2. Agent 评估：基于核心利益 + 性格 + 记忆 + 关键事件
   → 输出：{对手, 动作类型, 理由, 期望效果}

3. 代码仲裁器校验合法性 → 结算数值 → 写入事件流 → 更新状态

4. 若为第 1 轮 → 等待第 2 轮触发。
   若为第 2 轮 → 博弈结束，判断"召见"或"决策已定"。
```

### 2.4 同步博弈的冲突处理

可能出现秦同时向赵和齐施压，或赵齐同时在向对方求盟。仲裁器处理规则：

- **双向兼容**：赵求盟齐 + 齐也求盟赵 → 双方盟信↑↑（一拍即合）。
- **单向冲突**：秦离间赵齐 + 赵求盟齐 → 双方独立结算，取净值。
- **三方博弈**：秦在离间赵+齐，赵在求盟齐，齐在观望 → 仲裁器按动作时间戳加权。

## 三、三位君主 Agent 规格

### 3.1 秦王 · 嬴稷

| 维度 | 设定 |
|---|---|
| **核心利益** | 东出函谷吞六国。最怕：六国结成牢固合纵 |
| **性格** | 雄猜——果决但多疑，不轻信不妄动 |
| **同步博弈倾向** | 高频主动出击。每回合优先选择"向较弱/最犹豫的一方施压" |
| **可用博弈动作** | `军事施压`(目标国威↓ 己战心↑) / `遣使离间`(赵齐盟信↓) / `连横利诱`(许诺地/金 目标盟信↓向秦) / `备战蓄力`(己国威↑ 己战心↑) |
| **记忆** | 谁承诺了什么、谁在动摇、上次合纵被谁推动的 |
| **召见困局类型** | 展示实力后的威慑性提问 / 要求纵横家表态站队 / 试探赵齐联盟的虚实 |

### 3.2 赵王 · 赵何

| 维度 | 设定 |
|---|---|
| **核心利益** | 抗秦自保，但不愿独自承担秦的全部压力 |
| **性格** | 犹疑——想联齐又怕被拖累，想抗秦又怕激怒秦 |
| **同步博弈倾向** | 中等频率。优先"求盟＋观望"组合。 |
| **可用博弈动作** | `求盟联齐`(对齐盟信↑ 己盟信↑) / `备战固境`(己国威↑ 己战心↑) / `遣使试探`(获取他国真实意图) / `骑墙观望`(不做明确动作 己战心微↓) |
| **记忆** | 齐上次是否回应求援、秦最近的动作、谁在真心抗秦 |
| **召见困局类型** | "齐王说让我先动，我若先动秦必攻赵" / "平原君说联齐，廉颇说独战，孤该听谁的" |

### 3.3 齐王 · 田地

| 维度 | 设定 |
|---|---|
| **核心利益** | 渔利——维持秦赵均势，趁双方消耗壮大自己 |
| **性格** | 精明算计。口头答应但不出兵；谁给好处多就倾向谁 |
| **同步博弈倾向** | 低频观望。优先"等别人先动，再渔利" |
| **可用博弈动作** | `观望渔利`(己战心↓ 己盟信微↓) / `待价而沽`(接受利诱 目标国盟信↑向己) / `趁火打劫`(某方衰弱时施压) / `闭门自保`(拒绝外部影响) |
| **记忆** | 秦许了什么、赵给了什么条件、上次谁的许诺兑现了 |
| **召见困局类型** | "秦许我三城，赵求我结盟——孤该信谁" / "若我出兵，秦会不会先来打我" |

## 五、同步博弈的调度

### 5.1 轮次调度

```
自由行动阶段开始
  ├─ 立即（+1s）：三君主 Agent 同时启动第 1 轮 LLM 推理（并行请求）
  │   若 LLM 超时 5s → 回退 mock 规则决策表
  ├─ 各 Agent 陆续完成第 1 轮
  │   仲裁器校验 → 结算国家数值 → 写入事件流
  │   国家状态 = "谈判中·第1轮完成"
  │
  ├─ 第 1 轮全部完成后 +1s（无玩家干预）：自动进入第 2 轮
  │   或 玩家打牌 +0.5s：立即触发第 2 轮（重评估）
  │
  ├─ 第 2 轮：Agent 基于第 1 轮结果 + 玩家干预 → 做出最终决策
  │   仲裁器校验 → 结算 → 写入事件流
  │
  └─ 该国博弈完成 → 状态变为 "召见" 或 "决策已定"
      当所有三对博弈完成 → "[进入下回合]" 按钮亮起
```

### 5.2 召见与决策已定的判断

第 2 轮结束后，仲裁器判定该国状态：

- **召见**：该国博弈结果产生了**未决困境**——君主需要纵横家的建议才能下决心。玩家可前往打字面谈。
- **决策已定**：君主已做出明确决策，无需玩家介入。玩家仍可访问该国（查看决策结果 + 打牌试图推翻决策），但**不触发打字面谈**。

判断规则（代码仲裁器）：
- 若君主的第 2 轮裁决中包含"犹豫/两难/需要更多信息"标记 → "召见"。
- 若君主裁决为明确行动（如"出击""结盟""闭门"）→ "决策已定"。

### 5.3 玩家抵达"决策已定"国家的交互（v7.3 明确）

- 不弹面谈界面。
- 弹出该国决策摘要：_「齐王决定待价而沽，闭门观望——看来需要打破他的算盘」_
- 玩家可对该国打牌（游说/传信/许诺）→ **视为挑战其决策**：
  - 成功 → 推翻决策（该国状态 → 召见，玩家可前往面谈）。
  - 失败 → 名望 −3。
- 玩家不可对"决策已定"国家发起打字面谈。

### 5.4 Mock 决策表（降级）

每个君主配一张规则决策表：

```
状态输入：{本国三维, 对手三维, 近期事件标记, 关键事件}
  → 查表出：{目标国家, 动作类型, 幅度档位}
  → 拼接预制台词库中对应条目
```

Mock 模式下地图动画照常，事件流正常写入——玩家看到的博弈可视化不受影响，只是"台词"由预制库提供。

## 六、博弈可视化管线（从 Agent 输出到玩家视野）

```
Agent 输出 → 仲裁器结算 → 三路并发：
  ├─ 地图 view 更新：该国数值条动画变化 + 状态标签切换
  ├─ 事件流写入：一行文本描述此 Agent 的本轮动作
  └─ 召见触发：该国的"召见前情摘要"更新（供玩家面谈时查看）
```

玩家在地图上**实时看到**：
- 某国的国威条正在涨/跌（颜色闪烁 + 数值跳动）
- 两国之间使节图标移动中
- 国家状态标签从"谈判中"变成"**召见**"
- 事件流不断滚动，显示 Agent 之间的博弈动作

## 七、关键事件注入

每回合开始时，仲裁器根据脊骨段位 + 上回合数值状态生成**关键事件**（详见 [12 · 总体剧本](12-总体剧本.md)）。该关键事件以以下方式注入 Agent：

```
Agent Prompt 注入：
  本回合关键事件："{关键事件描述}"
  你的本轮决策应围绕此事件展开。
```

> 关键事件是**议题框架**，不是**剧本指令**。Agent 可围绕它自由选择动作，只要在博弈动作枚举范围内。
